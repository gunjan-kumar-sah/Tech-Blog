# 🤝 Contributing to Tech-Blog

First of all, **thank you for your interest in contributing to Tech-Blog!** 🎉

Your contributions help make this repository a better learning resource for students, developers, and placement aspirants around the world.

Whether you're fixing a typo, improving documentation, adding a new topic, or correcting an example—**every contribution is appreciated.**

---

# 🌟 Ways to Contribute

You can contribute in many ways, including:

* 📝 Improve existing documentation
* ➕ Add new technical articles
* 🐞 Fix spelling or grammatical mistakes
* 💡 Improve explanations with examples
* 🧠 Add shortcut tricks and interview tips
* ❓ Add practice questions or quizzes
* 🔗 Update learning resources
* 🎨 Improve formatting and readability

---

# 🚀 Getting Started

## 1️⃣ Fork the Repository

Click the **Fork** button on the top-right corner of the repository.

---

## 2️⃣ Clone Your Fork

```bash
git clone https://github.com/your-username/Tech-Blog.git
```

Replace `your-username` with your GitHub username.

---

## 3️⃣ Navigate to the Project

```bash
cd Tech-Blog
```

---

## 4️⃣ Create a New Branch

```bash
git checkout -b feature/add-new-topic
```

Choose a meaningful branch name.

Examples:

* `feature/python-notes`
* `feature/sql-guide`
* `docs/update-readme`
* `fix/typo-profit-loss`

---

## 5️⃣ Make Your Changes

Before submitting your work:

* Ensure the content is accurate.
* Follow Markdown formatting.
* Keep explanations simple and beginner-friendly.
* Test any code snippets if applicable.

---

## 6️⃣ Commit Your Changes

```bash
git add .

git commit -m "Add Time and Work practice questions"
```

Use clear and descriptive commit messages.

Good examples:

* `Fix typo in Percentage.md`
* `Improve README navigation`
* `Add Python file handling guide`
* `Update SQL examples`

---

## 7️⃣ Push Your Branch

```bash
git push origin feature/add-new-topic
```

---

## 8️⃣ Create a Pull Request

Go to your fork on GitHub and click **Compare & Pull Request**.

Please include:

* A clear title
* A brief description of your changes
* Screenshots (if applicable)

---

# 📋 Contribution Guidelines

Before submitting a Pull Request:

* ✅ Follow the existing folder structure.
* ✅ Use meaningful file names.
* ✅ Write beginner-friendly explanations.
* ✅ Add examples where appropriate.
* ✅ Check grammar and spelling.
* ✅ Avoid plagiarism—use your own words or properly attribute external references.
* ✅ Keep formatting consistent with the rest of the repository.

---

# 📝 Markdown Style Guide

Please follow these formatting rules:

* Use clear headings (`#`, `##`, `###`).
* Use bullet points for lists.
* Wrap code inside fenced code blocks.
* Keep tables readable.
* Add blank lines between sections for clarity.

Example:

````markdown
## Example

```python
print("Hello, World!")
```
````
